PRD: Decision Lab (Powered by MiroFish)

Generated by: Milo Date: 2026-04-10 Status: Draft (v2 -- rewritten to match full MiroFish vision) Requested by: Z Source: https://github.com/whitmorelabs/MiroFish

1. Goal

Give nonprofit leaders a full-fidelity simulation engine that constructs a parallel digital world from real-world context, populates it with thousands of intelligent agents (donors, community members, media, policymakers, staff, beneficiaries), and lets leaders test decisions across countless simulations before committing -- zero risk, total visibility.

2. Why

Nonprofit leaders make high-stakes decisions (comms campaigns, program pivots, policy positions, fundraising strategies) without knowing how their ecosystem will react. A press release could go viral or backfire. A program cut could trigger donor flight or community backlash. Today, leaders guess and pray.

Decision Lab changes this. Feed it real-world context (news articles, policy drafts, financial data, org documents), and it builds a living simulation of your stakeholder ecosystem -- thousands of agents with distinct personalities, opinions, and behavioral patterns -- then lets you inject your decision and watch what happens. Run it 100 times. See the range of outcomes. Make decisions after rehearsal, not on instinct.

Who it's for: Nonprofit executive directors, board members, comms directors, and program managers making decisions that affect communities, donors, and public perception.

Core technology: MiroFish -- our open-source multi-agent simulation engine built on CAMEL-AI OASIS, with Zep Cloud knowledge graphs for agent memory and social evolution.

3. Success Criteria

  • [ ] User can upload seed materials (PDFs, docs, news articles) that describe their decision context
  • [ ] System auto-generates a knowledge graph of entities and relationships from seed materials
  • [ ] System generates hundreds/thousands of agent profiles with distinct personalities, MBTI types, professions, interests, and stances
  • [ ] Agents simulate social interactions across platforms (posts, comments, likes, reposts, follows)
  • [ ] User can inject variables mid-simulation ("God's-eye view") to test specific decisions
  • [ ] Simulation produces a comprehensive report with predicted outcomes, consensus, risks, and recommendations
  • [ ] User can interview individual agents post-simulation to understand their reasoning
  • [ ] User can chat with the report agent for deeper analysis
  • [ ] Scenarios can be saved, compared, and revisited (decision history)
  • [ ] Simulation runs complete within 5-10 minutes for standard scenarios
  • [ ] Edify's 7 archetypes are integrated as expert commentators on simulation results
  • [ ] The feature is accessible to non-technical users with clear onboarding
  • [ ] Real-world seed data (news, policy) can be pulled live or uploaded manually

4. Tech Stack

  • Simulation engine: MiroFish (Python/Flask backend, Vue frontend) -- ported into Edify OS
  • Multi-agent framework: CAMEL-AI OASIS 0.2.5 (social media platform simulation)
  • Knowledge graph: Zep Cloud (entity/relationship management, temporal metadata, episode memory)
  • LLM: Claude Sonnet 4 via Anthropic API (BYOK) -- replacing Qwen-Plus from original MiroFish
  • Graph visualization: D3.js 7
  • Database: Supabase/PostgreSQL (scenario storage, simulation metadata)
  • Process management: Python subprocess for simulation execution
  • Frontend: Next.js (Edify OS) wrapping MiroFish Vue components or rewritten in React

5. The 5-Step Pipeline (from MiroFish)

Step 1: Seed & Graph Building

User uploads context materials (PDFs, articles, meeting notes, policy drafts) and describes what they want to simulate in natural language.

System: 1. Extracts and chunks text (configurable chunk size) 2. Generates ontology via LLM -- entity types (Person, Organization, MediaOutlet, DonorSegment, CommunityGroup) and relationship types (FUNDS, INFLUENCES, OPPOSES, REPRESENTS) 3. Builds knowledge graph in Zep Cloud with entities, edges, and temporal metadata

Step 2: Environment Setup

System reads entities from the graph and generates agent profiles: - Each entity becomes an agent with: age, gender, MBTI, profession, interests, activity levels - Simulation config generated by LLM: max rounds, activity schedules (timezone-aware), sentiment biases, stance assignments (supportive/opposing/neutral/observer), influence weights - User can review and adjust agent profiles and config before running

Step 3: Simulation

Agents interact on simulated social platforms in parallel: - Twitter simulation: CREATE_POST, LIKE, REPOST, QUOTE_POST, FOLLOW - Reddit simulation: CREATE_POST, COMMENT, LIKE, DISLIKE, SEARCH, TREND - Each round = simulated time period; agents act based on their personality, stance, and activity schedule - Information cascades emerge naturally (high-influence agents drive opinion formation) - Zep graph updated in real-time as agents act - User can inject variables mid-simulation via IPC ("What if this news breaks halfway through?")

Step 4: Report Generation

ReACT-pattern report agent analyzes the simulation: - Plans report structure (chapters/sections) - Calls Zep retrieval tools (InsightForge for deep analysis, PanoramaSearch for broad overview, QuickSearch for specific queries) - Generates comprehensive markdown report with predicted outcomes, risk analysis, and recommendations - Full action log available for transparency

Step 5: Deep Interaction

Post-simulation engagement: - Interview agents: Chat with any simulated agent to understand their reasoning and reactions - Chat with report: Ask follow-up questions to the report agent, which can call tools for additional analysis - Archetype commentary: Edify's 7 archetypes provide domain-specific analysis of the simulation results (Finance Director on budget impact, Marketing Director on comms implications, etc.)

6. Scope -- What is IN

  1. Full MiroFish 5-step pipeline integrated into Edify OS
  2. Seed material upload (PDF, MD, TXT) + natural language scenario description
  3. Ontology generation and knowledge graph construction via Zep Cloud
  4. Agent profile generation (hundreds to thousands of agents per simulation)
  5. Dual-platform parallel simulation (Twitter + Reddit style interactions)
  6. Real-time simulation monitoring with progress tracking and action logging
  7. God's-eye variable injection mid-simulation
  8. ReACT report generation with multi-tool Zep retrieval
  9. Agent interviews post-simulation
  10. Report chat for deeper analysis
  11. Archetype commentary layer -- Edify's 7 archetypes react to simulation results
  12. D3.js graph visualization of entity/relationship networks
  13. Scenario history -- save, compare, revisit past simulations
  14. Nonprofit-specific ontology templates (donors, beneficiaries, board members, funders, community groups, media)
  15. LLM swap from Qwen-Plus to Claude (Anthropic API, BYOK)

7. Scope -- What is OUT

  • Custom agent code execution (agents use LLM-driven behavior, not arbitrary code)
  • Real-time collaboration (multiple users in same simulation)
  • Integration with live social media feeds (simulated platforms only)
  • Voice input/output
  • Mobile-native app (web only)
  • Training/fine-tuning custom models per org

9. Key Decisions

Decision Choice Reason
Integration approach Port MiroFish into Edify OS as a service Same repo, shared auth/billing, seamless UX
LLM Claude Sonnet 4 (replacing Qwen-Plus) Consistent with Edify's BYOK model, better English output
Knowledge graph Zep Cloud (keep MiroFish's choice) Battle-tested in MiroFish, temporal metadata is critical for simulations
Simulation engine CAMEL-AI OASIS (keep) No reason to rebuild -- mature multi-agent framework
Frontend Rewrite Vue components in React/Next.js Consistent with Edify OS frontend stack
Graph visualization D3.js (keep) Industry standard, already proven in MiroFish
Nonprofit templates Pre-built ontology templates for common nonprofit scenarios Lower barrier to entry -- users don't start from scratch
Archetype integration Commentary layer on top of simulation results Best of both worlds -- MiroFish simulation + Edify expertise
Simulation subprocess Keep subprocess model Isolation, crash safety, resource management

10. Deployment Plan

  • New dependencies: camel-ai, camel-oasis, zep-cloud, pymupdf, d3 (frontend)

11. Handoff Notes

  • Start by forking MiroFish backend into services/decision-lab/ and adapting the LLM client from OpenAI SDK to Anthropic SDK
  • The ontology generator prompts need nonprofit-specific examples (donor types, funder relationships, community dynamics)
  • Agent profile generation should include nonprofit-relevant archetypes: major donor, small donor, board member, community leader, journalist, government official, beneficiary, volunteer, staff member
  • The IPC mechanism for God's-eye injection is partially built in MiroFish -- expand the UI to make it intuitive for non-technical users
  • Pre-build 3-5 scenario templates for first-time users: "Test a fundraising campaign", "Simulate response to a program closure", "Model community reaction to a policy change"
  • The archetype commentary layer should run AFTER the simulation completes -- each of the 7 Edify archetypes reads the report and provides their domain-specific take
  • CAMEL-AI OASIS requires Python 3.11+ -- verify compatibility with existing Edify backend
  • Zep Cloud has usage-based pricing -- build cost estimation into the UI so orgs know what a simulation will cost before running it
